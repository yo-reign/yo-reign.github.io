## Professional Self-Assessment

Going through the CS program at SNHU changed how I think about building software. When I started I just wrote code that worked, but over time I learned to think about maintainability, security, and picking the right tool for the problem. This capstone gave me a chance to put all of that together in one place.

On the collaboration and communication side, my job as an IT admin at a law firm taught me how to explain technical decisions to people who are not technical. Courses like CS-250 and CS-319 reinforced that. When it comes to data structures and algorithms, CS-300 gave me a strong foundation, but applying that knowledge in real projects is where it clicked. At my legal tech startup I built data migration pipelines where choosing the wrong structure meant slow processing, and in this capstone the reverse engineering project required scanning memory, traversing pointer chains, and doing projection math.

For software engineering and databases, I have been building real systems professionally for a while now. At Work Justice I manage a self-hosted stack with Docker, Headscale, and Caddy, and I have built internal tools like a lobby kiosk app and fleet automation with Ansible. Security has been a constant theme across the program and my work. CS-305, CS-405, and CS-410 all pushed the idea that security is a mindset, not a feature, and handling sensitive legal data at work reinforced that daily.

The three artifacts below show that I can work at different levels of the stack, from low-level memory work to cloud database integration, while keeping security in mind.
